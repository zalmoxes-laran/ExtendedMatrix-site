## What's new in 1.5

EM 1.5 stabilizes the axis of reconstructive steps that 1.4 had only
sketched, and lands the tools that were missing to live on top of it:
**Representation Models** become first-class citizens with their own
epoch-aware management, **proxy authoring** finally has a dedicated
suite where the geometry itself becomes paradata, **georeferencing**
enters the workflow as a passive layer, and **CronoFilter** + Landscape
mode let multi-graph scenes be navigated as a chronological narrative.
The release is the first one where the whole loop — sources, graph,
proxy, paradata, publication — runs end-to-end without manual jumps.

### <span class="wn-axis-tag">axis</span>Language

The EM language picks up the nodes and metadata that 1.4 still kept
implicit. Negative US categories now have their own canonical class
(`USN`), paradata gain `family` and `is_series` as additive metadata
per subtype (so the formalism stays at 1.5, no breaking changes), and
**Surface Areas** are promoted to first-class graph citizens via the
new Working Unit (UL) node type. Container-style group nodes (VSF /
US / USD) and the **dotted "transformation" connector** (`changed_from`)
give the graph the vocabulary to talk about instance chains and
reuse — including the new **Special Find (SF)** node for spolia.
Finally, a **`LocationNodeGroup`** for orthogonal spatial membership
(m:n, decoupled from time and intention) is in the pipeline as part
of the s3dgraphy 0.1.41 line, to land just after the 1.5.0 cut.

- **`USN` (Negative US)** as a canonical, palette-backed class — built
  on the s3Dgraphy v1.5.2 datamodel patch (additive `family` /
  `is_series` per subtype, no breaking changes).
- **Container groups** (VSF / US / USD) end-to-end across yEd,
  s3Dgraphy and EM Tools, with containment filtering and instance-chain
  edges (DP-36).
- **Transformation connector** (dotted `changed_from`) with GraphML
  round-trip and instance-chain filtering (DP-39).
- **Group nodes** formalised in the palette: Time Branch (green),
  Activity, Paradata (DP-43) — they now drive the Activity Manager's
  "By Activity" filter and the Heriverse per-source ParaData bundling.
- **Author / License / Embargo** classes with their own dashed
  connectors (`has_author`, `has_license`, `has_embargo`) and the
  hierarchical **propagative-metadata resolver** (node → swimlane →
  graph) covering chronology, author, license, embargo (DP-32, DP-51).
- **Special Find (SF)** node for re-used architectural / decorative
  elements (spolia) — the last development project before the 1.5
  cut (DP-26).
- **`LocationNodeGroup`** *(post-release, s3dgraphy 0.1.41)* —
  orthogonal m:n spatial membership; expressed at the graph layer
  before EM Tools UIs surface it.

### <span class="wn-axis-tag">axis</span>Tools

workflows. Where 1.4 had a single Stratigraphy panel and a fragmented
Representation Models story, 1.5 adds a **Document Manager** for the
documentary axis, an **RM Manager / RM Doc / RM Special Find** trio
for the reconstructive one, and a first **Proxy Box Creator** as the
opening tool of a planned proxy-creation suite. Distribution itself
moves to the **Blender Extension** format (`.zip`) with automatic
dependency management.

### <span class="wn-axis-tag">axis</span>Methodology

1.5 also changes *how* a project is modelled, not just *what* can be
modelled. The source list is no longer a flat collection of links: it
is a structured XLSX, mergeable into the graph through a wizard with
conflict resolution. Reconstructive steps are formalised in the graph
itself, instead of living as a side-document. **CronoFilter** turns
the chronological narrative into something the user navigates
interactively, and **Landscape mode** opens the door to scenographic
3D contexts that span multiple graphs in one Blender scene. From the
source list to the published proxy + paradata, the workflow is now
end-to-end without manual hand-offs.

- **CronoFilter** — chronological-horizons manager for multi-graph
  scenes; auto-generates horizons from all loaded graphs' epochs,
  saves to `.cf.json`, drives both the viewport and the Heriverse
  exporter (DP-10).
- **Landscape mode** — multi-graph scenes with per-graph publishable
  flag and graph-code object prefixing for territorial workflows.
- **Reconstructive steps formalised in the graph** — propagative
  metadata + container groups + the dotted transformation connector
  let the graph carry the steps explicitly, instead of relying on
  external commentary.
- **Source list as structured data** — XLSX template + Graph Merger,
  with a merge wizard for conflict resolution (DP-45).
- **Self-learning dataset** — an example dataset ships alongside the
  1.5.0 cut, so first-time users can step through a working graph
  without authoring one from scratch.

### Migrating from 1.4

EM 1.5 is **fully backward-compatible with 1.4**. There are no breaking changes — every graph authored against 1.4 continues to round-trip cleanly through 1.5 tooling, and the DP-26 spolia work landing in the cut maintains this guarantee.

What 1.5 introduces are **new and simpler ways to express the same concepts**, not replacements. The `ParadataNodeGroup` is the most visible example: it consolidates per-US paradata that previously hung as scattered nodes around each stratigraphic unit, producing a strong visual decluttering of the EM graph without removing any expressivity. Old-style scattered paradata still parses and renders correctly; new graphs benefit from the cleaner aggregation by default.

If you maintain in-flight 1.4 projects, no migration is required — open them in 1.5 tooling and continue authoring. The new constructs become available organically as you iterate.

> **Tip**: see [DP-XX — Paradata NodeGroup](https://dev.extendedmatrix.org/dev-projects/) for the formalization that drove the visual decluttering work.

  multi-graph [Heriverse](/tools/heriverse/) JSON export.
- **DP-11 — Heriverse Project** — exporter refactored into the
  `export_operators/heriverse/` subpackage; multi-graph JSON, GLTF +
  Draco + optional GPU instancing, separate texture pipeline,
  ParaData (RMDoc) and Special Finds export, plugin-style provider
  registry in the Export Manager panel.
- **DP-32 — Propagative Metadata System** — hierarchical scope
  resolver (node → swimlane → graph) with five built-in rules
  (chronology, author, license, embargo), bidirectional TPQ/TAQ
  closure, hard-policy paradox detection with claim attribution, and
  13 locked synthetic regression scenarios.
- **DP-35 — UI/UX Refactoring** — modular package architecture across
  every former monolithic top-level file, plugin-style Export Manager
  with `ExportProvider` registry, uniform experimental gating,
  Visual Manager with property-based colouring, Blender Extension
  packaging with multi-platform CI/CD.
- **DP-36 — Container Nodes** — VSF / US / USD container groups
  shipped end-to-end across yEd, s3Dgraphy and [EM Tools](/tools/em-tools/), with
  containment filtering and instance-chain edges; Anastylosis Manager
  refactored to consume the new container model.
- **DP-39 — Transformation Connector (Dotted)** — `changed_from` edge
  type with instance-chain helpers, GraphML round-trip preservation
  and instance-chain filtering in the Stratigraphy and RM Managers;
  absorbs the documentation work of the former DP-44.
- **DP-40 — Canvas Header Metadata Tags** — graph-level `[ID]`,
  `[ORCID]`, `[License]`, `[Embargo]` tags on the canvas title,
  consumed by the DP-32 resolver as graph-scope defaults.
- **DP-43 — Group Nodes** — Time Branch (green), Activity, and
  Paradata group nodes formalised in the palette and round-tripped
  end-to-end; the Activity Manager's "By Activity" filter and the
  Heriverse exporter's per-source ParaData bundling are driven by
  these groups.
- **DP-45 — XLSX Auxiliary Data & US Image Resources** — XLSX import
  and merge wizard with conflict resolution, paired with the s3Dgraphy
  Graph Merger (v0.1.33). US image resources and the Image Viewing
  System are deferred to 1.6.
- **DP-46 — Proxy Box Creator** — 7-point measurement system with the
  Extractor / Combiner paradata chain wrapped in a per-US
  `ParadataNodeGroup`, document-instance cloning at every run,
  GraphML write-lock guard, save-after-create on by default, and a
  chain-summary narrative in the panel.
- **DP-47 — RMDoc Manager (3D Document Manager)** — image import,
  quad and camera creation with DOSCO path resolution,
  master/instance model with certainty classification, three-axis
  Edit Classification operator, and an opt-in colour-by-geometry
  toggle that mirrors the document's geometry on the viewport quad.
- **DP-48 — GraphML Writer & Updater** — exporter + incremental
  patcher used by Create Host, Bake and Save GraphML; visual rules
  honoured end-to-end. Multi-source assembly pipeline split out to
  DP-53 for 1.6.
- **DP-50 — Surface Areale System** — projective and shrinkwrap
  surface-proxy strategies on Representation Models, with the full
  `US → Property → Extractor → Document → RM` paradata chain, an
  A/B/C complexity classifier, and the new Working Unit (UL) node
  type. Boolean + LOD strategy split out to DP-54 for 1.6.
- **DP-51 — Author Node Formalization** — palette-backed `AuthorNode`,
  `AuthorAINode`, `LicenseNode` and `EmbargoNode` classes with
  `has_author` / `has_license` / `has_embargo` edges, dashed-connector
  reclassification by target class, and integration with the DP-32
  resolver. *(Operational at the graph layer; surfacing in EM Tools
