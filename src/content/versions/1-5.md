> **Recommended for all new Extended Matrix projects.** EM 1.5 is the
> current Long-Term Support release — if you are unsure where to start,
> start here. EM 1.4 LTS remains available for legacy projects already
> authored against it.

## What's new in 1.5

EM 1.5 consolidates the formal language and the tools that speak it,
and becomes the recommended target for new projects. The headline this
round is **Representation Models management with advanced epoch
visualization** — every reconstructive object can now be tracked
across epochs through proper documentary anchors, with the paradata
kept aligned with the stratigraphic graph at every step.

### Headline

- **Representation Models management & advanced epoch visualization** —
  the **RM Manager**, **RM Doc** and **RM Special Find** nodes wire 3D
  reconstruction objects to the graph through documentary anchors. Each
  object resolves its own period, author and license through the
  propagative-metadata resolver, so chronological filters can drive the
  scene by epoch and the paradata behind every visible piece is one
  click away.

### Other highlights

- **Landscape mode** — manage multiple archaeological graphs in a
  single Blender scene, with per-graph publishable flag and graph-code
  prefixing for territorial workflows.
- **Chronofilter** — chronological-horizons manager for multi-graph
  scenes; auto-generates horizons from all loaded graphs' epochs, saves
  to `.cf.json`, and feeds the [Heriverse](/tools/heriverse/) exporter.
- **Document Manager** — a proper home for documentary sources:
  spatialise 2D evidence in 3D, classify it on three orthogonal axes
  (role / content nature / geometry), browse by completeness, and link
  it to the units it justifies.
- **Advanced proxy creation tools** — the **Proxy Box Creator** lands
  as the first of a planned suite: a 7-point measurement system where
  every point is recorded as an `ExtractorNode` and the seven are
  aggregated through a `CombinerNode`, so the proxy's geometry is
  itself paradata. **Surface Areas** join the same family for surface
  phenomena on Representation Models (projective and shrinkwrap
  strategies, with the new Working Unit / UL node type).
- **Stratigraphy Manager** — complete rewrite of the former US/USV
  Manager with containment filters, instance-chain tracking, an
  activity-based view, and a **single shared "Add US" dialog** used
  across every tool that creates units (Stratigraphy Manager, Proxy
  Box Creator, Surface Areas).
- **Anastylosis Manager (RMSF)** — link 3D objects to SpecialFind
  nodes with LOD management.
- **Georeferencing** — early infrastructure to bring scene shift and
  EPSG into the Extended Matrix workflow, in dialogue with BlenderGIS
  and the [3D Survey Collection](/tools/3dsc/). The graph carries a passive
  GeoPositionNode mirror in 1.5; richer reprojection capabilities are
  on the 1.6 roadmap.
- **3DSC 1.7.0** — the 3D Survey Collection matures into a fully
  featured Blender add-on with a companion [Metashape pipeline](/tools/3dsc-metashape/).
- **[s3dgraphy](/tools/s3dgraphy/) 1.5+** — first standalone release of the Python library.
  In EM 1.4 the code lived bundled inside [EM Tools](/tools/em-tools/); here it becomes
  its own component, published on PyPI.
- **Heriverse 1.5.0** — exporter refactored into a plugin-style
  architecture, with multi-graph JSON export, GLTF + Draco compression
  and optional GPU instancing, separate texture pipeline and ParaData
  export.
- Distribution as a **Blender Extension** (`.zip`) with automatic
  dependency management.

> **Migrating from 1.4?** Existing 1.4 projects can keep running on the
> 1.4 LTS line for as long as that release is supported. New projects
> should start from 1.5.

> **Self-learning dataset:** an **example dataset** ships alongside the
> 1.5.0 stable cut, so first-time users can step through a working graph
> without having to author one from scratch.

## Full changelog (draft)

*This changelog is a draft compiled from the development projects
tracker. For the canonical source see the [Extended Matrix dev
page](http://dev.extendedmatrix.org/dev-projects/).*

### Incorporated development projects

- **DP-07 — Document Node as a Spatio-Temporal Entity** — three-axis
  classification (role / content nature / geometry) formalised in the
  [yEd palette](/tools/yed-palette/), in [`s3Dgraphy`](/tools/s3dgraphy/) constructors, in the StratiMiner prompt
  and in the Document Manager Edit Classification operator.
- **DP-10 — Multigraph Project** — Landscape Mode, CronoFilter horizons
  with auto-from-epochs and `.cf.json` save/load, s3Dgraphy multi-graph
  API, per-graph publishable flag, graph-code object prefixing, and
  multi-graph [Heriverse](/tools/heriverse/) JSON export.
- **DP-11 — Heriverse Project** — exporter refactored into the
  `export_operators/heriverse/` subpackage; multi-graph JSON, GLTF +
  Draco + optional GPU instancing, separate texture pipeline,
  ParaData (RMDoc) and Special Finds export, plugin-style provider
  registry in the Export Manager panel.
- **DP-32 — Propagative Metadata System** — hierarchical scope
  resolver (node → swimlane → graph) with five built-in rules
  (chronology, author, license, embargo), bidirectional TPQ/TAQ
  closure, hard-policy paradox detection with claim attribution, and
  13 locked synthetic regression scenarios.
- **DP-35 — UI/UX Refactoring** — modular package architecture across
  every former monolithic top-level file, plugin-style Export Manager
  with `ExportProvider` registry, uniform experimental gating,
  Visual Manager with property-based colouring, Blender Extension
  packaging with multi-platform CI/CD.
- **DP-36 — Container Nodes** — VSF / US / USD container groups
  shipped end-to-end across yEd, s3Dgraphy and [EM Tools](/tools/em-tools/), with
  containment filtering and instance-chain edges; Anastylosis Manager
  refactored to consume the new container model.
- **DP-39 — Transformation Connector (Dotted)** — `changed_from` edge
  type with instance-chain helpers, GraphML round-trip preservation
  and instance-chain filtering in the Stratigraphy and RM Managers;
  absorbs the documentation work of the former DP-44.
- **DP-40 — Canvas Header Metadata Tags** — graph-level `[ID]`,
  `[ORCID]`, `[License]`, `[Embargo]` tags on the canvas title,
  consumed by the DP-32 resolver as graph-scope defaults.
- **DP-43 — Group Nodes** — Time Branch (green), Activity, and
  Paradata group nodes formalised in the palette and round-tripped
  end-to-end; the Activity Manager's "By Activity" filter and the
  Heriverse exporter's per-source ParaData bundling are driven by
  these groups.
- **DP-45 — XLSX Auxiliary Data & US Image Resources** — XLSX import
  and merge wizard with conflict resolution, paired with the s3Dgraphy
  Graph Merger (v0.1.33). US image resources and the Image Viewing
  System are deferred to 1.6.
- **DP-46 — Proxy Box Creator** — 7-point measurement system with the
  Extractor / Combiner paradata chain wrapped in a per-US
  `ParadataNodeGroup`, document-instance cloning at every run,
  GraphML write-lock guard, save-after-create on by default, and a
  chain-summary narrative in the panel.
- **DP-47 — RMDoc Manager (3D Document Manager)** — image import,
  quad and camera creation with DOSCO path resolution,
  master/instance model with certainty classification, three-axis
  Edit Classification operator, and an opt-in colour-by-geometry
  toggle that mirrors the document's geometry on the viewport quad.
- **DP-48 — GraphML Writer & Updater** — exporter + incremental
  patcher used by Create Host, Bake and Save GraphML; visual rules
  honoured end-to-end. Multi-source assembly pipeline split out to
  DP-53 for 1.6.
- **DP-50 — Surface Areale System** — projective and shrinkwrap
  surface-proxy strategies on Representation Models, with the full
  `US → Property → Extractor → Document → RM` paradata chain, an
  A/B/C complexity classifier, and the new Working Unit (UL) node
  type. Boolean + LOD strategy split out to DP-54 for 1.6.
- **DP-51 — Author Node Formalization** — palette-backed `AuthorNode`,
  `AuthorAINode`, `LicenseNode` and `EmbargoNode` classes with
  `has_author` / `has_license` / `has_embargo` edges, dashed-connector
  reclassification by target class, and integration with the DP-32
  resolver. *(Operational at the graph layer; surfacing in EM Tools
  UIs is incremental and continues into 1.6.)*
- **DP-55 — Stratigraphic Unit Creation Workflow Unification** — a
  single shared Add-US floating dialog used by Stratigraphy Manager,
  Proxy Box Creator and Surface Areas; JSON-driven US-type registry,
  gap-aware shared-pool numbering with legacy `PREFIX_ALIASES`, the
  canonical `USN` (Negative US) class on the s3Dgraphy side. Builds on
  the s3Dgraphy v1.5.2 datamodel patch (additive `family` + `is_series`
  metadata per subtype — the EM formalism stays at 1.5, no breaking
  changes).

### Pending before the 1.5 cut

- **DP-26 — Spolia Project** — last development project before the 1.5
  release: introduces the **Special Find (SF)** node with red border
  for re-used architectural and decorative elements (spolia),
  reinforcing the "US container" formalism in language and code. Lead:
  Nicola Delbarba (Ph.D. thesis).
